## Day 23 – Authorization & RBAC

# Overview

This module introduces Authorization and Role-Based Access Control (RBAC).
It demonstrates how access to system features can be restricted based on user roles.


# Authentication vs Authorization

Authentication → Who you are

Authorization → What you can do

# RBAC (Role-Based Access Control)

RBAC restricts access based on user roles.

Example Roles

    ->Admin

    ->Staff

# Role Access Mapping

Role	Access
Admin	Full CRUD
Staff	View + Update only

# Access Control Example

Admin → Can delete employee
