## Day 26 – Search Logic & Dynamic Queries

# Overview

This module implements search functionality and dynamic filtering to build flexible and efficient APIs.

# Learning Objectives

Implement search functionality
Use partial matching
Understand SQL LIKE queries
Combine multiple filters dynamically
Improve API flexibility

# Key Concepts

1.Search:

Finding data based on keyword input.

2.Partial Matching:

Matching part of a string instead of full value.

3.SQL LIKE:

Used for searching in databases:

SELECT * FROM employees WHERE name LIKE '%ram%';

# Features Implemented
Keyword-based search
Department filtering
Multi-filter combination
Dynamic query handling
