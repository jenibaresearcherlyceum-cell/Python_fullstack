## Day 24 – Validation & Secure Input Handling

# Overview

This module focuses on input validation and secure handling of user data.
It ensures that only valid and clean data is processed by the system.

# Learning Objectives

    ->Understand importance of input validation
    ->Validate user input properly
    ->Prevent invalid or incomplete data
    ->Handle API misuse gracefully
    ->Implement defensive programming

# Key Concepts

    Input Validation

        ->Checking whether input data meets required conditions.

    Data Sanitization

        ->Cleaning input data (e.g., removing spaces).

    Defensive Programming

        ->Assuming all user input can be invalid and handling it safely.

# Validation Rules Implemented

        ->Required field validation
        ->Data type checking
        ->String length validation
        ->Input sanitization

# Invalid Input Handling

## Example response:

  "status": "error",
  "message": "Invalid input"
